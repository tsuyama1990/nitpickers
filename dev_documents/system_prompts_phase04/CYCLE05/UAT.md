# CYCLE05 User Acceptance Testing

## Test Scenarios

### Scenario 1: Dynamic Execution Success Path
The objective of this scenario is to verify the successful dynamic execution of a passing UAT suite. The test involves a fully implemented, valid codebase and a corresponding, correctly defined Marimo UAT script in `tests/uat/`. The user initiates the complete pipeline. The system must invoke `uat_usecase.py`, dynamically execute the suite via `ProcessRunner`, and accurately capture the exit code 0. Crucially, the LangGraph must route the workflow directly to the finalization/PR generation stage, bypassing the Auditor completely. This Marimo scenario visually confirms that the pipeline can distinguish between success and failure without human intervention, ensuring that perfect code is automatically permitted to advance, thereby validating the core "happy path" of the automated testing framework.

### Scenario 2: Dynamic Execution Failure Routing
This scenario validates the critical failure path: dynamic execution encountering a broken UI and correctly routing to the Auditor. The test utilizes a codebase with a deliberate frontend defect that causes the Playwright test to fail. The user runs the pipeline. The system must execute the suite, capture the non-zero exit code, and successfully locate the screenshots generated by the CYCLE04 hook. The expected outcome is that `uat_usecase.py` packages these assets into the `UatExecutionState` and deterministically alters the LangGraph routing, forcibly sending the workflow to the `auditor_usecase` node for diagnosis, preventing any PR creation. This interactive Marimo test provides absolute proof that the "assumed success" vulnerability has been permanently closed, confirming the mechanical blockade operates flawlessly.

## Behavior Definitions
GIVEN a UAT suite executes successfully without UI or structural failures, WHEN the `uat_usecase.py` parses the `ProcessRunner` output, THEN it must construct a successful execution state AND the LangGraph router must definitively transition the workflow to the code submission phase.

GIVEN a UAT suite fails during dynamic execution, WHEN the `uat_usecase.py` receives the non-zero exit code, THEN it must automatically scan for and package the multi-modal artifacts (screenshots, traces) into the execution state AND the LangGraph router must immediately halt progression to submission, routing the state exclusively to the Stateless Auditor for diagnostic recovery, completely replacing subjective code review with mechanical blockade enforcement. This guarantees that no failed code escapes the local sandbox environment.
