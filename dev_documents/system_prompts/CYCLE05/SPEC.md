# CYCLE05 Specification

## Summary
The primary objective of CYCLE05 is to implement Phase 2: Dynamic Execution within the `uat_usecase.py` service. This cycle replaces the static "assumed success" bypass with an asynchronous, fully functional integration of the Pytest and Playwright execution suite. The goal is to construct a resilient pipeline that dynamically triggers the UATs within the sandbox, rigorously captures the execution state, and formally dictates workflow routing. Crucially, this cycle will implement the architectural refinement established in CYCLE04: instead of performing fragile directory scanning to find failure screenshots, `uat_usecase.py` will deterministically read and parse the explicitly generated `artifact_manifest.json`. This strictly typed JSON handoff ensures that the orchestrator accurately correlates specific test failures with their exact multi-modal payloads before routing the system to the Stateless Auditor for diagnostic recovery, cementing the mechanical blockade as the ultimate arbiter of quality.

## System Architecture
The system architecture for this cycle centers on the comprehensive overhaul of the `src/services/uat_usecase.py` module. The core mechanism is the `ProcessRunner`, instantiated to asynchronously execute `uv run pytest tests/uat/ --browser=chromium`. `uat_usecase.py` acts as the conductor of the "Behavioral Reality Sandbox." It must initiate the test execution asynchronously. Upon completion of the shell command, it parses the stdout, stderr, and the integer exit code. The critical architectural enhancement dictates that if the exit code is non-zero, the module must *not* scan directories; instead, it must explicitly read the `artifact_manifest.json` file generated by the Pytest hook. The output of this service is strictly defined by the enhanced `UatExecutionState` Pydantic model, which incorporates the parsed manifest data. If tests pass, the state dictionary signals the LangGraph router that the implementation is verified. If tests fail, the `UatExecutionState` object, fully populated with error logs and precisely mapped artifact paths from the JSON manifest, is injected into the state, signaling a mandatory routing transition to `auditor_usecase.py`. This deterministic, schema-driven handoff completely eliminates race conditions and hallucinated test results.

## Design Architecture
The design architecture relies heavily on the `UatExecutionState` Pydantic model to manage the complex data generated during a dynamic UI test failure. This model extends the base LangGraph state dictionary, strictly typing the properties governing the workflow's next steps. It includes `exit_code` (integer), `stdout` (string), `stderr` (string), and critically, an instance of the `TestReportManifest` schema (defined in CYCLE04). By enforcing `ConfigDict(extra='forbid')`, the schema guarantees that `uat_usecase.py` cannot inject arbitrary or malformed data. The validation logic within this model will verify that the paths parsed from the JSON manifest correspond to actual, readable files on disk before the state transition is permitted, acting as a final safety check. The primary consumer of this structured state is the LangGraph router (`routers.py`), which relies entirely and exclusively on the `exit_code` and the validated manifest within this object to make its next routing decision. This Schema-First design ensures that the transition from chaotic shell execution to the structured Python-based orchestrator is seamless, strongly typed, and entirely deterministic via explicit JSON contracts.

## Implementation Approach
The implementation plan for CYCLE05 starts with updating the `UatExecutionState` schema to accommodate the `TestReportManifest` data structure. Development then focuses on completely rewriting the primary execution method within `src/services/uat_usecase.py`. The hardcoded success bypass is removed. We will instantiate the `ProcessRunner` and configure it to run the Pytest suite. The asynchronous execution is awaited, and the resulting exit code and logs are captured. We will implement robust file reading logic within this method to explicitly load and parse the `artifact_manifest.json` file from disk. If a non-zero exit code is detected, the method will instantiate the `UatExecutionState` model, populating it with the shell logs and the strictly validated JSON manifest object. This state object is then securely merged into the primary graph state. Finally, the conditional routing logic within `src/nodes/routers.py` must be explicitly verified to ensure it correctly reads the newly structured state and branches reliably to the auditor node upon failure, utilizing the data strictly mapped by the JSON handoff.

## Test Strategy

### Unit Testing Approach
The unit testing strategy focuses intensely on the isolated behavior of the rewritten `uat_usecase.py`. We will use `unittest.mock.patch` to heavily mock the `ProcessRunner` execution and `builtins.open` for file system operations, preventing actual shell or browser execution. We simulate scenarios where `ProcessRunner` returns a successful exit code (0), verifying the use case constructs a passing `UatExecutionState` without attempting to read a manifest. We then simulate a failure (non-zero exit code), mocking the file system to return a valid JSON string representing the `TestReportManifest`. The tests must assert that the use case successfully parses this JSON string, instantiates the `UatExecutionState`, and correctly maps the artifact paths. We will rigorously test error handling; if the process fails but the manifest JSON is malformed or missing (e.g., a critical pre-hook failure), the use case must gracefully handle this, log a severe warning, and still produce a valid error state without crashing the agentic framework. All mocked objects must be strictly type-hinted for `mypy`.

### Integration Testing Approach
The integration testing strategy for CYCLE05 requires verifying the complex interaction between `uat_usecase.py`, the `ProcessRunner`, the JSON manifest, and the LangGraph state dictionary. We construct a temporary directory containing both a passing and a deliberately failing UAT script. We programmatically invoke `uat_usecase.py`. For the passing script, we assert the resulting state dictionary reflects a `0` exit code. For the failing script, we verify that Pytest execution occurred via the sandbox, that the `conftest.py` hook successfully generated the physical `.png` artifact AND the `artifact_manifest.json` file. Crucially, we assert that `uat_usecase.py` successfully read and parsed that exact JSON file, populated the `UatExecutionState` with the accurate file paths, and updated the graph state dictionary with the non-zero exit code. This end-to-end integration test is paramount, proving the system has successfully replaced assumed success with a fully functional, multi-modal gatekeeper driven by explicit JSON contracts.
