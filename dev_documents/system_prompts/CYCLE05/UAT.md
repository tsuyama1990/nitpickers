# CYCLE05 User Acceptance Testing

## Test Scenarios

### Scenario 1: Dynamic Execution Success Path
The objective of this scenario is to verify the successful dynamic execution of a passing UAT suite. The test involves a fully implemented, valid codebase and a corresponding, correctly defined Marimo UAT script in `tests/uat/`. The user initiates the complete pipeline. The system must invoke `uat_usecase.py`, dynamically execute the suite via `ProcessRunner`, and accurately capture the exit code 0. Crucially, the LangGraph must route the workflow directly to the finalization/PR generation stage, bypassing the Auditor completely, and verifying that no manifest parsing exceptions occur when no manifest is expected. This Marimo scenario visually confirms that the pipeline can distinguish between success and failure deterministically, ensuring that perfect code is automatically permitted to advance.

### Scenario 2: Dynamic Execution Failure Routing via JSON Manifest
This scenario validates the critical failure path: dynamic execution encountering a broken UI and correctly routing to the Auditor using the precise JSON manifest. The test utilizes a codebase with a deliberate frontend defect. The user runs the pipeline. The system must execute the suite and capture the non-zero exit code. The expected outcome is that `uat_usecase.py` explicitly locates, opens, and parses the `artifact_manifest.json` generated by the hook. It packages these strictly mapped assets into the `UatExecutionState` and deterministically alters the LangGraph routing, forcibly sending the workflow to the `auditor_usecase` node for diagnosis, preventing PR creation. This interactive Marimo test provides absolute proof that failures are reliably tracked and routed based on explicit JSON handoffs, not guesswork.

## Behavior Definitions
GIVEN a UAT suite executes successfully without UI or structural failures, WHEN the `uat_usecase.py` parses the `ProcessRunner` output, THEN it must construct a successful execution state, bypass manifest parsing, AND the LangGraph router must definitively transition the workflow to the code submission phase.

GIVEN a UAT suite fails during dynamic execution, WHEN the `uat_usecase.py` receives the non-zero exit code, THEN it must explicitly read and parse the `TestReportManifest.json` file from disk, package the precisely mapped multi-modal artifacts into the execution state, AND the LangGraph router must immediately halt progression to submission, routing the state exclusively to the Stateless Auditor for diagnostic recovery based on the verified JSON data.
