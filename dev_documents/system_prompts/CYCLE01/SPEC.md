# CYCLE01: Implement 5-Phase Architecture Redesign

## Summary
The goal of CYCLE01 is to rigorously implement the complete 5-Phase Architecture outlined in the `SYSTEM_ARCHITECTURE.md`. This upgrade refactors the existing monolithic LangGraph structure into distinct phases (Init, Architect, Coder, Integration, and UAT). The `src/state.py` will be enriched with new typed variables (`is_refactoring`, `current_auditor_index`, `audit_attempt_count`), and `src/graph.py` will be overhauled to define `_create_coder_graph` (Phase 2), `_create_integration_graph` (Phase 3), and adjust the `_create_qa_graph` (Phase 4).

The core of Phase 3, the integration phase, relies on the `master_integrator_node` utilizing a 3-Way Diff generated by `src/services/conflict_manager.py`. It is crucial that the Integration phase is triggered only after all Phase 2 parallel cycles have successfully concluded, which requires orchestrating modifications in `src/cli.py` and `src/services/workflow.py`. The transition ensures zero side-effects between parallel environments and guarantees that UAT runs only on an integrated base. This document specifies the exact schemas, constraints, implementation steps, and strict testing requirements. To validate the pipeline's stability, comprehensive unit and integration testing using mock API configurations will be enforced.

## System Architecture
This cycle involves modifying the existing repository structure to reflect the new 5-Phase strategy.

```text
src/
├── **state.py**                # Define `is_refactoring`, `current_auditor_index` in CycleState, and `IntegrationState`
├── **graph.py**                # Define _create_coder_graph, _create_integration_graph, _create_qa_graph
├── **cli.py**                  # Orchestrate run_pipeline to trigger sequential parallel and integration execution
├── nodes/
│   ├── **routers.py**          # Define route_sandbox_evaluate, route_auditor, route_final_critic
│   └── **integrations.py**     # Ensure `master_integrator_node` uses the 3-Way Diff context
└── services/
    ├── **conflict_manager.py** # Implement `build_conflict_package` using Git 3-Way Diff (`git show :1:file`, `:2:file`, `:3:file`)
    ├── **uat_usecase.py**      # Decouple from Phase 2 to only run after Phase 3 completion
    └── **workflow.py**         # Control execution flow of Parallel Coder Graphs -> Integration Graph
```

The execution flow begins with Phase 0, moving to the Architect. Phase 2 spans multiple cycles in parallel. Each cycle interacts with the `CycleState` strictly bounded by its scope. The `is_refactoring` boolean and `current_auditor_index` integer govern the internal loops. Upon completion, Phase 3 synchronizes the outputs via `git_merge_node` and potentially `master_integrator_node`. `conflict_manager.py` manages the construction of a 3-way payload. Phase 4 initiates only after the global sandbox validations pass.

## Design Architecture
The Pydantic-based schema dictates system state explicitly, serving as the blueprint for LangGraph operations.

- **`CycleState` (in `src/state.py`)**:
  - Adds `is_refactoring: bool = False`, `current_auditor_index: int = 1`, and `audit_attempt_count: int = 0`.
  - *Constraints*: Types must strictly adhere to Pydantic validation rules.
  - *Expected Consumers*: `route_sandbox_evaluate`, `route_auditor`, `route_final_critic`.
- **`IntegrationState` (in `src/state.py`)**:
  - Encapsulates target files, conflict blobs, Base, Local (Cycle), and Remote code contexts.
  - *Constraints*: Must prevent directory traversal when validating paths.
- **`routers.py`**:
  - Contains strictly typed routing functions evaluating `CycleState`. E.g., `route_auditor` checks `current_auditor_index` against a maximum threshold before yielding `"pass_all"`.
- **`conflict_manager.py`**:
  - Utilizes `subprocess.run` asynchronously to extract exact Git markers for 3-Way diff. Expected producers are Git conflicts; consumers are the Master Integrator LLM prompts.

## Implementation Approach
1. **Extend Domain Models**: Start by modifying `src/state.py`. Add the required fields to `CycleState` using standard type hints compatible with Pydantic 2.x. Establish the `IntegrationState` definition.
2. **Refactor Graph Routings**: Update `src/nodes/routers.py` to include `route_sandbox_evaluate`, `route_auditor`, and `route_final_critic` based on the values of `is_refactoring` and `current_auditor_index`.
3. **Overhaul Workflow Graphs**: In `src/graph.py`, refactor `_create_coder_graph` to utilize the new routers, chaining `auditor_node` serially up to 3 times before invoking the `refactor_node`. Construct the `_create_integration_graph` mapping the 3-Way Diff integration logic.
4. **Implement 3-Way Diff Contextualization**: In `src/services/conflict_manager.py`, rewrite `build_conflict_package`. Use `git show :1:<file>` for Base, `:2:<file>` for Local, and `:3:<file>` for Remote. Structure the returned prompt explicitly containing these three distinct blocks.
5. **Phase Orchestration**: In `src/cli.py` and `src/services/workflow.py`, rewrite the high-level execution loops. Use `asyncio.gather` for parallel Phase 2 execution, await their completion, and subsequently trigger the Phase 3 `build_integration_graph`.

## Test Strategy
The test strategy mandates strict adherence to the **Sandbox Resilience Rule**. Every test simulating LLM integration or API dependency MUST implement mock patching.

### Unit Testing Approach
- **State Validation**: Instantiate `CycleState` explicitly omitting new variables and verify default initializations. Instantiate with invalid types (e.g., string for `current_auditor_index`) to assert Pydantic throws corresponding errors.
- **Routing Logic**: In `tests/unit/test_routers.py`, construct mock `CycleState` objects dictating various conditions (e.g., `is_refactoring=True`, `sandbox_status="failed"`). Validate that `route_sandbox_evaluate` returns `"final_critic"`, `"failed"`, or `"auditor"` deterministically.
- **Conflict Package Generation**: Mock the `subprocess` interactions in `src/services/conflict_manager.py`. Assert that `build_conflict_package` correctly interpolates the Base, Local, and Remote mock outputs into the designated Markdown structure.

### Integration Testing Approach
- **Workflow Orchestration**: In `tests/integration/test_workflow.py`, simulate a 2-cycle run. Mock the internal graphs (`build_coder_graph`, `build_integration_graph`) to return immediate success states. Validate that `workflow.py` fires Phase 2 in parallel and exclusively calls Phase 3 only upon complete success of Phase 2.
- **3-Way Diff Git Simulation**: Initialize a local temporary bare repository. Create a programmatic merge conflict spanning Base, Branch A, and Branch B. Invoke `build_conflict_package` natively without mocking Git. Verify the output prompt contains the real conflicting snippets separated correctly. No LLMs are called; external APIs are mocked completely using `pytest-mock` to avoid retry loops.
- **DB Rollback Rule**: If executing state tracking requiring persistence, encapsulate within a rolled-back transaction fixture.

Additional Architecture details:
The `src/services/uat_usecase.py` logic must be completely decoupled from Phase 2 to ensure Phase 4 executes exclusively after Phase 3. The `src/cli.py` and `src/services/workflow.py` must orchestrate `run_pipeline` by executing parallel Coder sessions and waiting for their `END` states before firing the sequential `master_integrator_node`. The integration pipeline relies strictly on Git commands. The `build_conflict_package` must construct prompts formatted as:

```python
### Base (元のコード)
{base_code}
### Branch A の変更 (Local)
{local_code}
### Branch B の変更 (Remote)
{remote_code}
```

The system architecture defines the file structure explicitly. Modification involves adapting existing schemas in `src/state.py` and routing nodes in `src/nodes/routers.py`. The updated files govern the core execution path for all parallel `_create_coder_graph` instances. The entire implementation will focus purely on architectural routing without altering any external tools or existing core LLM instructions aside from those specified.
