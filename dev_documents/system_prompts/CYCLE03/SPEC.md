# CYCLE03 SPEC: GitHub Write Operations & Jules Orchestration Migration

## Summary
The final and most critical phase, CYCLE03, focuses on migrating the high-risk, state-mutating operations: GitHub Write capabilities and Jules Session Orchestration. This cycle completely deprecates the complex legacy logic within `src/services/git/` and `src/services/jules/`. By binding destructive tools (like `push_commit`, `create_pull_request`, `create_session`, and `review_changes`) exclusively to the `Master Integrator` and `Global Refactor` nodes via the `McpClientManager`, the system effectively eliminates fragile custom Python middleware handling Git merge conflicts and parallel agent synchronization. The LangGraph engine now relies entirely on the `@modelcontextprotocol/server-github` and `@google/jules-mcp` sidecars for these robust, cloud-native operations.

## System Architecture
This cycle completes the "MCP Router" pattern. The `McpClientManager` now manages three concurrent Node.js sidecars via Stdio: E2B, GitHub, and Jules. The architecture explicitly enforces the Principle of Least Privilege established in CYCLE02. The `Master Integrator` node is the *only* component granted access to the GitHub Write tools. The `Global Refactor` node is the *only* component granted access to the Jules Orchestration tools. This strict boundary management prevents hallucinatory code execution or unintended repository mutations. Once these nodes are successfully refactored to utilize the native MCP `ToolCalls`, the legacy service directories (`src/services/git/` and `src/services/jules/`) are permanently deleted.

## Design Architecture
The focus shifts to correctly formatting and dispatching the raw intents of the `Master Integrator` and `Global Refactor` into the standard JSON-RPC schemas expected by the GitHub and Jules MCP servers.

**Domain Concepts & Core Mappings:**
-   **GitHub Writes:** A commit is no longer a localized `subprocess` execution tracking local branch state. It is a native `push_commit` ToolCall sent directly to the GitHub API via the MCP server. Branching and Pull Request creation are handled symmetrically. The `CycleState` must be structurally updated to natively store remote identifiers like `pull_request_url` and `remote_commit_hash` instead of `local_branch_name`.
-   **Jules Orchestration:** A parallel session is no longer a manual HTTP polling loop managed by `jules_session_state.py`. It is a discrete `create_session` ToolCall, with the Jules MCP server natively managing the fleet dispatch and returning the reconciled diffs upon completion.

**Key Invariants & Constraints:**
1.  **Atomicity of Commits:** The `Master Integrator` node must construct `push_commit` tool calls that accurately represent the entire changeset generated by the `Coder` node within the `CycleState`, ensuring no partial code is pushed to remote branches.
2.  **Strict Fleet Synchronization (Race Condition Locks):** The `Global Refactor` node handles diffs returned by the parallel fleets orchestrated by Jules MCP. To prevent state corruption, the node must not blindly append diffs to the context. Instead, it must utilize a rigid, sequential patching lock mechanism (e.g., via strict parsing into deterministic `FilePatch` Pydantic models) to evaluate incoming `ToolMessage` diffs sequentially before applying them to the `CycleState`.
3.  **Complete Deprecation:** The `src/services/git/` and `src/services/jules/` directories must be entirely removed, confirming that zero legacy dependency remains.

## Implementation Approach
1.  **Finalize McpClientManager:** Update `src/services/mcp_client_manager.py` to concurrently connect to `npx -y @google/jules-mcp`, securely passing the `JULES_API_KEY`. Implement `get_write_tools(server_name)` and `get_orchestration_tools(server_name)` to safely expose these toolsets.
2.  **Refactor Master Integrator:** Modify `src/nodes/master_integrator.py`. Remove all legacy `git_ops` imports. Inject the `McpClientManager`, retrieve the GitHub Write tools (`push_commit`, `create_pull_request`), and bind them to the LLM. Update prompts to instruct the LLM to execute commits autonomously based on the `CycleState`.
3.  **Refactor Jules Nodes:** Modify `src/nodes/global_refactor.py` and `src/nodes/audit_orchestrator.py`. Remove legacy `jules_client` imports. Retrieve and bind the Jules Orchestration tools (`create_session`, `review_changes`). Update prompts to instruct the LLM to dispatch agents autonomously.
4.  **Execute Deletion:** Delete the `src/services/git/` and `src/services/jules/` directories entirely. Remove all references in tests and config files.
5.  **State Transformation & Pydantic Updates:** Update the `CycleState` Pydantic model definition to ensure fields formally map to remote Git states (e.g., `pr_url: HttpUrl | None`) rather than legacy local process outputs. Ensure the `Master Integrator` accurately extracts these values from the `ToolMessage`.
6.  **Concurrency Locks:** Implement a sequential merging algorithm inside the `Global Refactor` node that applies locking validation when parsing multiple session diffs returned simultaneously from the Jules MCP, preventing race condition file corruption.

## Test Strategy

**Integration Testing Approach:**
The primary testing focus must be on executing end-to-end integration tests in a secure, isolated repository and fleet environment.
-   **GitHub Write Robustness:** Utilize `tests/ac_cdd/integration/test_git_robustness.py`. Point the system at an isolated dummy repository. Provide a `CycleState` containing significant code changes and require the `Master Integrator` to securely construct a `push_commit` tool call and subsequently a `create_pull_request` tool call via the GitHub MCP server. The test must definitively assert the existence of the remote branch and PR.
-   **Jules Fleet Dispatch:** Execute `tests/ac_cdd/integration/test_mcp_jules_session_dispatch.py` (New File). Trigger a scenario requiring the `Global Refactor` node. Assert that the `create_session` tool is successfully invoked via the Jules MCP server, dispatches dummy agents (if test mode allows), and correctly returns a reconciled `ToolMessage` session diff without dropping the LangGraph state.
-   **Mechanical Gate Validation:** Execute `tests/unit/test_cycle03_mechanical_gate.py`. Attempt to invoke a `push_commit` action from the `Auditor` node. Assert that the tool execution is firmly rejected or omitted from the toolset, mechanically enforcing the Principle of Least Privilege.
